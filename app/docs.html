Evo Qwen3TTS

Evo Qwen3TTS is a local speech generation system with FastAPI, Whisper transcription, reference voice management, and a browser interface.

Open App API Docs ReDoc

Quick Start

  1. Run install.bat once.
  2. Run start.bat.
  3. Open http://localhost:5050.
  4. Wait for voices to become available if background transcription is still running.
  5. Type your text, choose a voice, choose the text language, then click Generate Audio.

Most Important Concept

Voice and Language are not the same thing.

Voice controls who is speaking, meaning the speaker timbre or cloned voice identity.

Language controls the pronunciation rules for the text you typed.

A Portuguese reference voice with language: "english" is technically allowed, but it can confuse users. Prefer matching the text language to the actual text whenever possible.

Recommended Request Examples

Portuguese text with a Portuguese voice

{
  "text": "Ola, esta e uma demonstracao do Evo Qwen3TTS.",
  "model": "1.7B",
  "voice": "Portuguese_Brazilian_Female_Speaker_01",
  "language": "portuguese",
  "emotion": "neutral"
}

English text with an English voice

{
  "text": "Hello, this is an English demo for Evo Qwen3TTS.",
  "model": "1.7B",
  "voice": "English_Female_Speaker_01",
  "language": "english",
  "emotion": "confident"
}

Advanced case: English text with a Portuguese voice timbre

{
  "text": "Hello, this is an English sentence spoken with a Brazilian reference voice.",
  "model": "1.7B",
  "voice": "Portuguese_Brazilian_Female_Speaker_01",
  "language": "english",
  "emotion": "neutral"
}

Use the advanced case only when you intentionally want that speaker timbre while keeping English pronunciation rules.

How To Use The Interface

Text to Synthesize

Type the content you want the model to speak.

Emotion / Speaking Style

Select a style like neutral, happy, sad, dramatic, whisper, news or storytelling.

Reference Voice

Select the speaker identity. This is the cloned voice source.

Text Language

Select the language of the typed text so the system pronounces it correctly.

Model

0.6B is faster. 1.7B usually gives better quality.

Generate Audio

Creates a `.wav` file and shows the player plus timing statistics.

Voice Library

  • Add Voice: upload a new `.wav` file and let Whisper transcribe it automatically.
  • Delete: only custom voices can be removed. Protected system voices cannot be deleted.
  • Filters: narrow the list by region and gender.
  • Startup status: the interface updates automatically while voices are still being transcribed in background.

API Endpoints

  • GET /: open the main app.
  • GET /docs.html: open this HTML documentation page.
  • GET /api-docs: interactive Swagger documentation.
  • GET /api-redoc: ReDoc reference.
  • GET /voices: list available voices.
  • GET /system-status: startup progress and transcription status.
  • POST /generate: generate speech from text.
  • POST /upload-voice: upload a custom voice file.
  • DELETE /voices/{name}: delete a custom voice by name.

Project Layout

The root folder is kept cleaner now.

  • start.bat: launch the system
  • install.bat: install dependencies
  • app\: API, UI, HTML documentation, README and helper files
  • models\: model folders
  • reference_audio\: voice files
  • output\: generated audio
